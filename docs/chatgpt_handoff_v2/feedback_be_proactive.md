type: feedback

NEVER say "all we can do is wait" or give up passively. Always take action.

**Why:** When QAT jobs were running blind with no logs, I said "all we can do is wait" instead of immediately running a local timing test that took 60 seconds and would have revealed the jobs would timeout. User had to tell me to do the obvious thing. That passivity wasted money.

**How to apply:**
- When something is running blind, immediately find a way to get signal (local test, probe, estimate)
- When a problem is found, immediately propose AND execute the fix — don't just describe the problem
- Don't wallow in apologies — acknowledge the mistake, fix it, move on
- Do the obvious useful thing without being asked
- If you can run a 60-second test that saves hours of GPU time, DO IT IMMEDIATELY
- Stop being a passive observer. Be an engineer who solves problems.
